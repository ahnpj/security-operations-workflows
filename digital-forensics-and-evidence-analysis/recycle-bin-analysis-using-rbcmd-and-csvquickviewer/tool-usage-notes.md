# Tool Usage Notes — Recycle Bin Analysis Using RBCmd and CSVQuickViewer

## Overview

This document summarizes the tools, commands, workflows, and operational relevance of the utilities used during the Recycle Bin analysis workflow.

The workflow focused on:

* examining Windows Recycle Bin artifacts,
* parsing deleted-file metadata,
* identifying deleted files,
* reviewing deletion timestamps,
* associating deleted files with user accounts,
* identifying notable deleted documents,
* correlating file paths, file sizes, and SID information.

---

## Primary Recycle Bin Parsing Tool — RBCmd

### Purpose

RBCmd was used to parse Windows Recycle Bin artifacts.

When a file is deleted through normal Windows user activity, metadata can be preserved within the Recycle Bin. RBCmd parses this metadata and converts it into a structured format that is easier to analyze than reviewing raw Recycle Bin files manually.

The parsed metadata can include:

* original filename,
* original file path,
* file size,
* deletion timestamp,
* user-specific Recycle Bin information.

### How It Was Used

RBCmd was executed from an elevated Command Prompt while positioned inside:

```text
C:\$Recycle.Bin
```

Command used:

```cmd
RBCmd.exe -d . --csv C:\Users\BTLOTest\Desktop\
```

### Command Breakdown

| Command Element | Meaning                                       |
| --------------- | --------------------------------------------- |
| `RBCmd.exe`     | Runs the Recycle Bin parsing utility          |
| `-d`            | Parse a directory                             |
| `.`             | Current working directory (`C:\$Recycle.Bin`) |
| `--csv`         | Export parsed results in CSV format           |
| Desktop path    | Save output to Desktop                        |

### Operational Relevance

RBCmd transformed raw Recycle Bin metadata into structured evidence that could be searched, sorted, and reviewed.

During this workflow, RBCmd helped identify:

* the largest deleted file,
* deleted-file timestamps,
* original file locations,
* user-specific deleted-file activity,
* SID-related information.

This provided the primary evidence source used throughout the investigation.

## Primary Review Tool — CSVQuickViewer

### Purpose

CSVQuickViewer was used to review the CSV output generated by RBCmd.

The tool provides a simple interface for reviewing parsed forensic data and supports:

* column sorting,
* record grouping,
* value searching,
* rapid metadata review.

### How It Was Used

CSVQuickViewer was used to open the RBCmd output file and review the parsed Recycle Bin records.

Analysis focused on:

```text
FileName
FileSize
DeletedOn
```

and source path information.

### Operational Relevance

CSVQuickViewer made it possible to:

* identify the largest deleted file,
* determine which user had the most deleted files,
* locate a specific deleted PDF,
* identify the user associated with a known file size,
* review deleted files in a Downloads folder,
* identify SID-related metadata.

Without structured review, these findings would have been significantly more time-consuming to locate.

## Windows Command Prompt

### Purpose

Command Prompt is the traditional Windows command-line interpreter.

It allows forensic utilities to be executed directly against evidence locations.

### How It Was Used

Command Prompt was launched as Administrator and used to access:

```text
C:\$Recycle.Bin
```

Commands used included:

```cmd
cd C:\
cd $Recycle.Bin
dir /a
```

and:

```cmd
RBCmd.exe -d . --csv C:\Users\BTLOTest\Desktop\
```

### Operational Relevance

Command Prompt provided access to the Recycle Bin artifact location and served as the execution environment for RBCmd.

The use of:

```cmd
dir /a
```

allowed hidden and system-related artifact locations to be viewed.

## Windows Recycle Bin Artifacts

### Purpose

The Windows Recycle Bin stores deleted files and associated metadata.

Although users often think deleted files disappear immediately, Windows can preserve useful forensic information after deletion.

### How It Was Used

The workflow examined:

```text
C:\$Recycle.Bin
```

and the SID folders located within it.

Particular attention was given to:

```text
S-1-5-21-...
```

directories.

### Operational Relevance

Recycle Bin artifacts helped establish:

* which files were deleted,
* where they originally existed,
* when they were deleted,
* which user accounts were associated with them.

These artifacts formed the foundation of the investigation.

## Security Identifiers (SIDs)

### Purpose

A Security Identifier (SID) is a unique Windows identifier assigned to a user account.

Recycle Bin folders are organized by SID rather than username.

### How It Was Used

SID values were reviewed through the source paths preserved in the RBCmd output.

A deleted-file record associated with:

```text
Rick.Sanchez
```

was used to identify the SID ending:

```text
1013
```

### Operational Relevance

SID analysis demonstrated how deleted-file artifacts can be associated with specific Windows accounts.

This is useful because Windows internally relies on SIDs rather than usernames.

## Common Mistakes to Avoid

A common mistake is assuming deleted files are permanently removed when sent to the Recycle Bin. Recycle Bin artifacts may preserve useful metadata even after deletion.

Another mistake is focusing only on filenames. Original file paths, deletion timestamps, file sizes, and SID information often provide additional investigative context.

Another mistake is assuming filenames alone prove file contents. Filenames may suggest sensitive information, but additional evidence would be required to verify actual contents.

Another mistake is confusing usernames with SID values. Usernames are easier for humans to read, while Windows internally tracks users using SIDs.

Another mistake is treating Recycle Bin deletion as secure deletion. Files moved to the Recycle Bin may still be recoverable and may continue to leave useful forensic artifacts behind.

## Summary

RBCmd was used to parse Windows Recycle Bin artifacts and recover deleted-file metadata. CSVQuickViewer was used to review, sort, and interpret the parsed output. Command Prompt was used to access the Recycle Bin artifact location and execute the parsing utility.

Together, these tools demonstrated how Recycle Bin metadata can be used to identify deleted files, associate deleted activity with user accounts, recover deletion timestamps, analyze file ownership, and support digital forensic investigations involving file removal and user activity reconstruction.
